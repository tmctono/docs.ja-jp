## <a name="nullable-references-and-static-analysis"></a><span data-ttu-id="ee420-123">Null 許容参照とスタティック分析</span><span class="sxs-lookup"><span data-stu-id="ee420-123">Nullable references and static analysis</span></span>

<span data-ttu-id="ee420-124">前のセクションの例は、Null 許容参照型の特性を示しています。</span><span class="sxs-lookup"><span data-stu-id="ee420-124">The examples in the previous section illustrate the nature of nullable reference types.</span></span> <span data-ttu-id="ee420-125">Null 許容参照型は新しいクラスの型ではなく、既存の参照型に対する注釈です。</span><span class="sxs-lookup"><span data-stu-id="ee420-125">Nullable reference types aren't new class types, but rather annotations on existing reference types.</span></span> <span data-ttu-id="ee420-126">コンパイラは、これらの注釈を使用して、ユーザーがコード内で発生する可能性のある Null 参照エラーを検出しやすくします。</span><span class="sxs-lookup"><span data-stu-id="ee420-126">The compiler uses those annotations to help you find potential null reference errors in your code.</span></span> <span data-ttu-id="ee420-127">Null 非許容参照型と Null 参照型の間には、ランタイムの違いはありません。</span><span class="sxs-lookup"><span data-stu-id="ee420-127">There's no runtime difference between a non-nullable reference type and a nullable reference type.</span></span> <span data-ttu-id="ee420-128">コンパイラは、Null 非許容参照型のランタイム チェックを追加しません。</span><span class="sxs-lookup"><span data-stu-id="ee420-128">The compiler doesn't add any runtime checking for non-nullable reference types.</span></span> <span data-ttu-id="ee420-129">この利点は、コンパイル時分析にあります。</span><span class="sxs-lookup"><span data-stu-id="ee420-129">The benefits are in the compile-time analysis.</span></span> <span data-ttu-id="ee420-130">コンパイラは、コード内で発生する可能性のある Null エラーの検出と修正に役立つ警告を生成します。</span><span class="sxs-lookup"><span data-stu-id="ee420-130">The compiler generates warnings that help you find and fix potential null errors in your code.</span></span> <span data-ttu-id="ee420-131">意図を宣言すると、コンパイラは、コードがその意図に違反したときに警告を発行します。</span><span class="sxs-lookup"><span data-stu-id="ee420-131">You declare your intent, and the compiler warns you when your code violates that intent.</span></span>

<span data-ttu-id="ee420-132">Null 許容の有効なコンテキストでは、コンパイラは、Null 許容と Null 非許容の両方の参照型の変数に対してスタティック分析を実行します。</span><span class="sxs-lookup"><span data-stu-id="ee420-132">In a nullable enabled context, the compiler performs static analysis on variables of any reference type, both nullable and non-nullable.</span></span> <span data-ttu-id="ee420-133">コンパイラは、各参照変数の null 状態を "*null ではない*" または "*null かもしれない*" として追跡します。</span><span class="sxs-lookup"><span data-stu-id="ee420-133">The compiler tracks the null state of each reference variable as either *not null* or *maybe null*.</span></span> <span data-ttu-id="ee420-134">Null 非許容参照の既定の状態は、"*null ではない*" です。</span><span class="sxs-lookup"><span data-stu-id="ee420-134">The default state of a non-nullable reference is *not null*.</span></span> <span data-ttu-id="ee420-135">Null 許容参照の既定の状態は、"*null かもしれない*" です。</span><span class="sxs-lookup"><span data-stu-id="ee420-135">The default state of a nullable reference is *maybe null*.</span></span>

<span data-ttu-id="ee420-142">Null 許容参照型は、`null`に初期化することも、割り当てることもできます。</span><span class="sxs-lookup"><span data-stu-id="ee420-142">Nullable reference types may be initialized or assigned to `null`.</span></span> <span data-ttu-id="ee420-143">したがって、スタティック分析では、変数が逆参照される前に、それが "*null ではない*" であることを確認する必要があります。</span><span class="sxs-lookup"><span data-stu-id="ee420-143">Therefore, static analysis must determine that a variable is *not null* before it's dereferenced.</span></span> <span data-ttu-id="ee420-144">Null 許容参照が "*null かもしれない*" であると判断された場合、それを Null 非許容参照変数に割り当てることはできません。</span><span class="sxs-lookup"><span data-stu-id="ee420-144">If a nullable reference is determined to be *maybe null*, it can't be assigned to a non-nullable reference variable.</span></span> <span data-ttu-id="ee420-145">次のクラスは、これらの警告の例を示します。</span><span class="sxs-lookup"><span data-stu-id="ee420-145">The following class shows examples of these warnings:</span></span>
