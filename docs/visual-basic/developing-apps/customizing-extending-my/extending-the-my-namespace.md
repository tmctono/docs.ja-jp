- <span data-ttu-id="fcc84-161">**拡張コードを分離します。**</span><span class="sxs-lookup"><span data-stu-id="fcc84-161">**Isolate the extension code.**</span></span> <span data-ttu-id="fcc84-162">コードを1つのファイルに配置すると、拡張機能が Visual Studio 項目テンプレートとして簡単に展開できるようになります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-162">Placing the code in a single file makes your extension easily deployable as a Visual Studio item template.</span></span> <span data-ttu-id="fcc84-163">詳細については、このトピックで後述する「拡張機能のパッケージ化と展開」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="fcc84-163">For more information, see "Packaging and Deploying Extensions" later in this topic.</span></span> <span data-ttu-id="fcc84-164">すべての `My` 名前空間拡張コードを1つのファイルまたはプロジェクト内の別のフォルダーに配置すると、ユーザーは `My` 名前空間の拡張機能を見つけることができます。</span><span class="sxs-lookup"><span data-stu-id="fcc84-164">Placing all the `My` namespace extension code in a single file or a separate folder in a project will also help users locate the `My` namespace extension.</span></span>

## <a name="designing-class-libraries-for-my"></a><span data-ttu-id="fcc84-165">@No__t_0 のクラスライブラリの設計</span><span class="sxs-lookup"><span data-stu-id="fcc84-165">Designing class libraries for `My`</span></span>

<span data-ttu-id="fcc84-166">ほとんどのオブジェクトモデルの場合と同様に、設計パターンの中には `My` 名前空間で適切に機能するものもあれば、そうでないものもあります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-166">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="fcc84-167">@No__t_0 名前空間の拡張機能を設計するときは、次の原則を考慮してください。</span><span class="sxs-lookup"><span data-stu-id="fcc84-167">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="fcc84-168">**ステートレスメソッド。**</span><span class="sxs-lookup"><span data-stu-id="fcc84-168">**Stateless methods.**</span></span> <span data-ttu-id="fcc84-169">@No__t_0 名前空間のメソッドは、特定のタスクに完全なソリューションを提供する必要があります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-169">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="fcc84-170">メソッドに渡されたパラメーター値が、特定のタスクを完了するために必要なすべての入力を提供していることを確認します。</span><span class="sxs-lookup"><span data-stu-id="fcc84-170">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="fcc84-171">リソースへのオープン接続など、以前の状態に依存するメソッドを作成しないようにします。</span><span class="sxs-lookup"><span data-stu-id="fcc84-171">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>
- <span data-ttu-id="fcc84-172">**グローバルインスタンス。**</span><span class="sxs-lookup"><span data-stu-id="fcc84-172">**Global instances.**</span></span> <span data-ttu-id="fcc84-173">@No__t_0 名前空間で保持される状態は、プロジェクトに対してグローバルです。</span><span class="sxs-lookup"><span data-stu-id="fcc84-173">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="fcc84-174">たとえば、`My.Application.Info` は、アプリケーション全体で共有される状態をカプセル化します。</span><span class="sxs-lookup"><span data-stu-id="fcc84-174">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>
- <span data-ttu-id="fcc84-175">**単純なパラメーターの型。**</span><span class="sxs-lookup"><span data-stu-id="fcc84-175">**Simple parameter types.**</span></span> <span data-ttu-id="fcc84-176">複雑なパラメーターの型を避けることで、シンプルなものを維持します。</span><span class="sxs-lookup"><span data-stu-id="fcc84-176">Keep things simple by avoiding complex parameter types.</span></span> <span data-ttu-id="fcc84-177">代わりに、パラメーター入力を受け取らないか、文字列、プリミティブ型などの単純な入力型を受け取るメソッドを作成します。</span><span class="sxs-lookup"><span data-stu-id="fcc84-177">Instead, create methods that either take no parameter input or that take simple input types such as strings, primitive types, and so on.</span></span>
- <span data-ttu-id="fcc84-178">**ファクトリメソッド。**</span><span class="sxs-lookup"><span data-stu-id="fcc84-178">**Factory methods.**</span></span> <span data-ttu-id="fcc84-179">一部の型は、インスタンス化が困難な場合があります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-179">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="fcc84-180">ファクトリメソッドを `My` 名前空間の拡張機能として指定すると、このカテゴリに分類される型をより簡単に検出して使用できます。</span><span class="sxs-lookup"><span data-stu-id="fcc84-180">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="fcc84-181">適切に機能するファクトリメソッドの例としては、`My.Computer.FileSystem.OpenTextFileReader` があります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-181">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="fcc84-182">.NET Framework で使用できるストリームの種類はいくつかあります。</span><span class="sxs-lookup"><span data-stu-id="fcc84-182">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="fcc84-183">具体的には、テキストファイルを指定することによって、使用するストリームをユーザーが理解できるように `OpenTextFileReader` します。</span><span class="sxs-lookup"><span data-stu-id="fcc84-183">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>

<span data-ttu-id="fcc84-184">これらのガイドラインでは、クラスライブラリの一般的な設計原則は使用できません。</span><span class="sxs-lookup"><span data-stu-id="fcc84-184">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="fcc84-185">これらは、Visual Basic と `My` 名前空間を使用している開発者向けに最適化された推奨事項です。</span><span class="sxs-lookup"><span data-stu-id="fcc84-185">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="fcc84-186">クラスライブラリを作成するための一般的な設計原則については、「[フレームワークのデザインガイドライン](../../../standard/design-guidelines/index.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="fcc84-186">For general design principles for creating class libraries, see [Framework Design Guidelines](../../../standard/design-guidelines/index.md).</span></span>
