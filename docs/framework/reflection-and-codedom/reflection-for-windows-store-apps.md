<span data-ttu-id="2a93c-103">.NET Framework 4.5 以降、.NET Framework には、Windows 8.x ストアアプリで使用する一連のリフレクション型とメンバーが含まれています。</span><span class="sxs-lookup"><span data-stu-id="2a93c-103">Starting with the .NET Framework 4.5, the .NET Framework includes a set of reflection types and members for use in Windows 8.x Store apps.</span></span> <span data-ttu-id="2a93c-104">これらの型およびメンバーは、完全な .NET Framework だけでなく Windows ストア アプリ用 .NET でも使用できます。</span><span class="sxs-lookup"><span data-stu-id="2a93c-104">These types and members are available in the full .NET Framework as well as in the .NET for Windows Store apps.</span></span> <span data-ttu-id="2a93c-105">このドキュメントでは、これらと .NET Framework 4 以前のバージョンでの対応するものとの主な相違点について説明します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-105">This document explains the major differences between these and their counterparts in the .NET Framework 4 and earlier versions.</span></span>

 <span data-ttu-id="2a93c-109">[!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] では、<xref:System.Reflection.TypeInfo> クラスに .NET Framework 4 <xref:System.Type> クラスの機能の一部が含まれます。</span><span class="sxs-lookup"><span data-stu-id="2a93c-109">In the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)], the <xref:System.Reflection.TypeInfo> class contains some of the functionality of the .NET Framework 4 <xref:System.Type> class.</span></span> <span data-ttu-id="2a93c-110"><xref:System.Type> オブジェクトは型定義への参照を表し、<xref:System.Reflection.TypeInfo> オブジェクトは型定義自体を表します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-110">A <xref:System.Type> object represents a reference to a type definition, whereas a <xref:System.Reflection.TypeInfo> object represents the type definition itself.</span></span> <span data-ttu-id="2a93c-111">これによって、参照するアセンブリをランタイムが必ずしも読み込まなくても、<xref:System.Type> オブジェクトを操作できるようになります。</span><span class="sxs-lookup"><span data-stu-id="2a93c-111">This enables you to manipulate <xref:System.Type> objects without necessarily requiring the runtime to load the assembly they reference.</span></span> <span data-ttu-id="2a93c-112">関連付けられた <xref:System.Reflection.TypeInfo> オブジェクトを取得すると、アセンブリが強制的に読み込まれます。</span><span class="sxs-lookup"><span data-stu-id="2a93c-112">Getting the associated <xref:System.Reflection.TypeInfo> object forces the assembly to load.</span></span>  
  
 <span data-ttu-id="2a93c-113"><xref:System.Reflection.TypeInfo> には <xref:System.Type> で使用できるメンバーの多くが含まれ、[!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] のリフレクションのプロパティの多くは <xref:System.Reflection.TypeInfo> オブジェクトのコレクションを返します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-113"><xref:System.Reflection.TypeInfo> contains many of the members available on <xref:System.Type>, and many of the reflection properties in the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] return collections of <xref:System.Reflection.TypeInfo> objects.</span></span> <span data-ttu-id="2a93c-114"><xref:System.Reflection.TypeInfo> オブジェクトから <xref:System.Type> オブジェクトを取得するには、<xref:System.Reflection.IReflectableType.GetTypeInfo%2A> メソッドを使用します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-114">To get a <xref:System.Reflection.TypeInfo> object from a <xref:System.Type> object, use the <xref:System.Reflection.IReflectableType.GetTypeInfo%2A> method.</span></span>  
  
## <a name="query-methods"></a><span data-ttu-id="2a93c-115">クエリ メソッド</span><span class="sxs-lookup"><span data-stu-id="2a93c-115">Query Methods</span></span>  
 <span data-ttu-id="2a93c-116">[!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] では、配列を返すメソッドではなく、<xref:System.Collections.Generic.IEnumerable%601> コレクションを返すリフレクション プロパティを使用します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-116">In the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)], you use the reflection properties that return <xref:System.Collections.Generic.IEnumerable%601> collections instead of methods that return arrays.</span></span> <span data-ttu-id="2a93c-117">リフレクション コンテキストは、大型のアセンブリまたは型用に、これらのコレクションの限定的な走査を実装できます。</span><span class="sxs-lookup"><span data-stu-id="2a93c-117">Reflection contexts can implement lazy traversal of these collections for large assemblies or types.</span></span>  
  
 <span data-ttu-id="2a93c-118">リフレクション プロパティは、継承ツリーを走査する代わりに、特定オブジェクトの宣言されたメソッドのみを返します。</span><span class="sxs-lookup"><span data-stu-id="2a93c-118">The reflection properties return only the declared methods on a particular object instead of traversing the inheritance tree.</span></span> <span data-ttu-id="2a93c-119">また、フィルター処理に <xref:System.Reflection.BindingFlags> パラメーターを使用しません。</span><span class="sxs-lookup"><span data-stu-id="2a93c-119">Moreover, they do not use <xref:System.Reflection.BindingFlags> parameters for filtering.</span></span> <span data-ttu-id="2a93c-120">代わりに、返されるコレクションで LINQ クエリを使用することにより、フィルター処理をユーザー コード内で発生させます。</span><span class="sxs-lookup"><span data-stu-id="2a93c-120">Instead, filtering takes place in user code, by using LINQ queries on the returned collections.</span></span> <span data-ttu-id="2a93c-121">(`typeof(Object)` の結果などにより) ランタイムから始まるリフレクション オブジェクトの場合、継承ツリーを走査するには <xref:System.Reflection.RuntimeReflectionExtensions> クラスのヘルパー メソッドを使用するのが最も適切です。</span><span class="sxs-lookup"><span data-stu-id="2a93c-121">For reflection objects that originate with the runtime (for example, as the result of `typeof(Object)`), traversing the inheritance tree is best accomplished by using the helper methods of the <xref:System.Reflection.RuntimeReflectionExtensions> class.</span></span> <span data-ttu-id="2a93c-122">カスタマイズされたリフレクション コンテキストのオブジェクトのコンシューマーは、これらのメソッドを使用できず、独自に継承ツリーを走査する必要があります。</span><span class="sxs-lookup"><span data-stu-id="2a93c-122">Consumers of objects from customized reflection contexts cannot use these methods, and must traverse the inheritance tree themselves.</span></span>
