<span data-ttu-id="cc64d-103">特定の条件に基づいてデータを並べ替え、UI コントロールを介してそのデータをクライアントに提供する機能は、データ バインディングの重要な特徴です。</span><span class="sxs-lookup"><span data-stu-id="cc64d-103">The ability to sort data based on specific criteria and then present the data to a client through a UI control is an important aspect of data binding.</span></span> <span data-ttu-id="cc64d-104"><xref:System.Data.DataView> には、データを並べ替え、特定の並べ替え条件に従って並べ替えられたデータ行を返す方法がいくつか用意されています。</span><span class="sxs-lookup"><span data-stu-id="cc64d-104"><xref:System.Data.DataView> provides several ways to sort data and return data rows ordered by specific ordering criteria.</span></span> <span data-ttu-id="cc64d-105">文字列ベースの並べ替え機能に加えて、<xref:System.Data.DataView> では、並べ替え条件として統合言語クエリ (LINQ) 式を使用することもできます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-105">In addition to its string-based sorting capabilities, <xref:System.Data.DataView> also enables you to use Language-Integrated Query (LINQ) expressions for the sorting criteria.</span></span> <span data-ttu-id="cc64d-106">LINQ 式では、文字列ベースの並べ替えよりもはるかに複雑で強力な並べ替え操作が可能です。</span><span class="sxs-lookup"><span data-stu-id="cc64d-106">LINQ expressions allow for much more complex and powerful sorting operations than string-based sorting.</span></span> <span data-ttu-id="cc64d-107">このトピックでは、<xref:System.Data.DataView> を使用して並べ替えを行うこの 2 つの方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="cc64d-107">This topic describes both approaches to sorting using <xref:System.Data.DataView>.</span></span>  
  
## <a name="creating-dataview-from-a-query-with-sorting-information"></a><span data-ttu-id="cc64d-108">並べ替え情報を含むクエリによる DataView の作成</span><span class="sxs-lookup"><span data-stu-id="cc64d-108">Creating DataView from a Query with Sorting Information</span></span>  
 <span data-ttu-id="cc64d-109"><xref:System.Data.DataView> オブジェクトは、LINQ to DataSet クエリから作成できます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-109">A <xref:System.Data.DataView> object can be created from a LINQ to DataSet query.</span></span> <span data-ttu-id="cc64d-110">このクエリに <xref:System.Linq.Enumerable.OrderBy%2A>、<xref:System.Linq.Enumerable.OrderByDescending%2A>、<xref:System.Linq.Enumerable.ThenBy%2A>、または <xref:System.Linq.Enumerable.ThenByDescending%2A> 句が含まれている場合、これらの句の式は、<xref:System.Data.DataView>内のデータを並べ替える基準として使用されます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-110">If that query contains an <xref:System.Linq.Enumerable.OrderBy%2A>, <xref:System.Linq.Enumerable.OrderByDescending%2A>, <xref:System.Linq.Enumerable.ThenBy%2A>, or <xref:System.Linq.Enumerable.ThenByDescending%2A> clause the expressions in these clauses are used as the basis for sorting the data in the <xref:System.Data.DataView>.</span></span> <span data-ttu-id="cc64d-111">たとえば、クエリに `Order By…`句と `Then By…` 句が含まれている場合、結果として得られる <xref:System.Data.DataView> では、指定された両方の列によってデータが順序付けられます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-111">For example, if the query contains the `Order By…`and `Then By…` clauses, the resulting <xref:System.Data.DataView> would order the data by both columns specified.</span></span>  
  
 <span data-ttu-id="cc64d-112">式ベースの並べ替えは、文字列ベースの並べ替えよりもはるかに強力で複雑な並べ替え機能を提供します。</span><span class="sxs-lookup"><span data-stu-id="cc64d-112">Expression-based sorting offers more powerful and complex sorting than the simpler string-based sorting.</span></span> <span data-ttu-id="cc64d-113">文字列ベースの並べ替え機能と式ベースの並べ替え機能は、相互に排他的です。</span><span class="sxs-lookup"><span data-stu-id="cc64d-113">Note that string-based and expression-based sorting are mutually exclusive.</span></span> <span data-ttu-id="cc64d-114"><xref:System.Data.DataView.Sort%2A> をクエリから作成した後に文字列ベースの <xref:System.Data.DataView> を設定した場合、クエリから推論される式ベースの並べ替えはクリアされます (再設定できません)。</span><span class="sxs-lookup"><span data-stu-id="cc64d-114">If the string-based <xref:System.Data.DataView.Sort%2A> is set after a <xref:System.Data.DataView> is created from a query, the expression-based filter inferred from the query is cleared and cannot be reset.</span></span>  
  
 <span data-ttu-id="cc64d-115"><xref:System.Data.DataView> のインデックスは、<xref:System.Data.DataView> の作成時に構築されるほか、並べ替えまたはフィルター処理の情報が変更されたときにも構築されます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-115">The index for a <xref:System.Data.DataView> is built both when the <xref:System.Data.DataView> is created and when any of the sorting or filtering information is modified.</span></span> <span data-ttu-id="cc64d-116"><xref:System.Data.DataView> を作成する LINQ to DataSet クエリに並べ替え条件を指定し、後で並べ替え情報を変更しないようにすることで、最高のパフォーマンスを得ることができます。</span><span class="sxs-lookup"><span data-stu-id="cc64d-116">You get the best performance by supplying sorting criteria in the LINQ to DataSet query that the <xref:System.Data.DataView> is created from and not modifying the sorting information, later.</span></span> <span data-ttu-id="cc64d-117">詳細については、「 [DataView のパフォーマンス](dataview-performance.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="cc64d-117">For more information, see [DataView Performance](dataview-performance.md).</span></span>  
  
> [!NOTE]
> <span data-ttu-id="cc64d-118">ほとんどの場合、並べ替えに使用する式は、副作用のない確定的な式である必要があります。</span><span class="sxs-lookup"><span data-stu-id="cc64d-118">In most cases, the expressions used for sorting should not have side effects and must be deterministic.</span></span> <span data-ttu-id="cc64d-119">また、並べ替え処理は任意の回数実行されるため、特定の実行回数に依存するロジックが式に含まれないようにしてください。</span><span class="sxs-lookup"><span data-stu-id="cc64d-119">Also, the expressions should not contain any logic that depends on a set number of executions, because the sorting operations might be executed any number of times.</span></span>
