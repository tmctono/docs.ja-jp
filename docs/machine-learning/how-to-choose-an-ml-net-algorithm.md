<span data-ttu-id="d9305-104">[ML.NET タスク](resources/tasks.md) ごとに選択できるトレーニング アルゴリズムは複数あります。</span><span class="sxs-lookup"><span data-stu-id="d9305-104">For each [ML.NET task](resources/tasks.md), there are multiple training algorithms to choose from.</span></span> <span data-ttu-id="d9305-105">どれを選択するかは、解決しようとしている問題、データの特性、利用できるコンピューティング リソースとストレージ リソースによって変わります。</span><span class="sxs-lookup"><span data-stu-id="d9305-105">Which one to choose depends on the problem you are trying to solve, the characteristics of your data, and the compute and storage resources you have available.</span></span> <span data-ttu-id="d9305-106">機械学習モデルのトレーニングは反復処理である点に注意することが重要です。</span><span class="sxs-lookup"><span data-stu-id="d9305-106">It is important to note that training a machine learning model is an iterative process.</span></span> <span data-ttu-id="d9305-107">場合によっては、最適なものを見つけるために複数のアルゴリズムを試す必要があります。</span><span class="sxs-lookup"><span data-stu-id="d9305-107">You might need to try multiple algorithms to find the one that works best.</span></span>

<span data-ttu-id="d9305-108">アルゴリズムは**特徴**に対して動作します。</span><span class="sxs-lookup"><span data-stu-id="d9305-108">Algorithms operate on **features**.</span></span> <span data-ttu-id="d9305-109">特徴は、入力データから計算された数値です。</span><span class="sxs-lookup"><span data-stu-id="d9305-109">Features are numerical values computed from your input data.</span></span> <span data-ttu-id="d9305-110">それらは機械学習アルゴリズムに最適な入力です。</span><span class="sxs-lookup"><span data-stu-id="d9305-110">They are optimal inputs for machine learning algorithms.</span></span> <span data-ttu-id="d9305-111">生の入力データを 1 つ以上の[データ変換](resources/transforms.md)を使用して特徴に変換します。</span><span class="sxs-lookup"><span data-stu-id="d9305-111">You transform your raw input data into features using one or more [data transforms](resources/transforms.md).</span></span> <span data-ttu-id="d9305-112">たとえば、テキスト データは、単語数と単語の組み合わせ数のセットに変換されます。</span><span class="sxs-lookup"><span data-stu-id="d9305-112">For example, text data is transformed into a set of word counts and word combination counts.</span></span> <span data-ttu-id="d9305-113">データ変換を使用して生のデータの種類から特徴が抽出されると、**特徴付けされた**と呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="d9305-113">Once the features have been extracted from a raw data type using data transforms, they are referred to as **featurized**.</span></span> <span data-ttu-id="d9305-114">たとえば、特徴付けされたテキスト、特徴付けされた画像データなどです。</span><span class="sxs-lookup"><span data-stu-id="d9305-114">For example, featurized text, or featurized image data.</span></span>

## <a name="trainer--algorithm--task"></a><span data-ttu-id="d9305-115">トレーナー = アルゴリズム + タスク</span><span class="sxs-lookup"><span data-stu-id="d9305-115">Trainer = Algorithm + Task</span></span>

<span data-ttu-id="d9305-116">アルゴリズムは、**モデル**を生成するために実行される数学です。</span><span class="sxs-lookup"><span data-stu-id="d9305-116">An algorithm is the math that executes to produce a **model**.</span></span> <span data-ttu-id="d9305-117">アルゴリズムが異なると、特性が異なるモデルが生成されます。</span><span class="sxs-lookup"><span data-stu-id="d9305-117">Different algorithms produce models with different characteristics.</span></span> 

<span data-ttu-id="d9305-118">ML.NET では、同じアルゴリズムを異なるタスクに適用できます。</span><span class="sxs-lookup"><span data-stu-id="d9305-118">With ML.NET, the same algorithm can be applied to different tasks.</span></span> <span data-ttu-id="d9305-119">たとえば、二項分類、多クラス分類と回帰の確率的デュアル連携のとれたアセントを使用できます。</span><span class="sxs-lookup"><span data-stu-id="d9305-119">For example, Stochastic Dual Coordinated Ascent can be used for Binary Classification, Multiclass Classification, and Regression.</span></span> <span data-ttu-id="d9305-120">違いは、タスクに合わせてアルゴリズムの出力が解釈される方法にあります。</span><span class="sxs-lookup"><span data-stu-id="d9305-120">The difference is in how the output of the algorithm is interpreted to match the task.</span></span> 

<span data-ttu-id="d9305-121">ML.NET には、各アルゴリズムとタスクの組み合わせに対して、トレーニング アルゴリズムを実行し、解釈するコンポーネントが用意されています。</span><span class="sxs-lookup"><span data-stu-id="d9305-121">For each algorithm/task combination, ML.NET provides a component that executes the training algorithm and does the interpretation.</span></span> <span data-ttu-id="d9305-122">これらのコンポーネントはトレーナーと呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="d9305-122">These components are called trainers.</span></span> <span data-ttu-id="d9305-123">たとえば、<xref:Microsoft.ML.Trainers.SdcaRegressionTrainer> には、**回帰**タスクに適用される **StochasticDualCoordinatedAscent** アルゴリズムが使用されます。</span><span class="sxs-lookup"><span data-stu-id="d9305-123">For example, the <xref:Microsoft.ML.Trainers.SdcaRegressionTrainer> uses the **StochasticDualCoordinatedAscent** algorithm applied to the **Regression** task.</span></span>

## <a name="linear-algorithms"></a><span data-ttu-id="d9305-124">線形アルゴリズム</span><span class="sxs-lookup"><span data-stu-id="d9305-124">Linear algorithms</span></span>

<span data-ttu-id="d9305-125">線形アルゴリズムでは、入力データと一連の**重み**の線形結合から**スコア**を計算するモデルが生成されます。</span><span class="sxs-lookup"><span data-stu-id="d9305-125">Linear algorithms produce a model that calculates **scores** from a linear combination of the input data and a set of **weights**.</span></span> <span data-ttu-id="d9305-126">重みは、トレーニング中に推定されるモデルのパラメーターです。</span><span class="sxs-lookup"><span data-stu-id="d9305-126">The weights are parameters of the model estimated during training.</span></span>

<span data-ttu-id="d9305-127">線形アルゴリズムは、[線形分離可能](https://en.wikipedia.org/wiki/Linear_separability)である特徴の場合に適しています。</span><span class="sxs-lookup"><span data-stu-id="d9305-127">Linear algorithms work well for features that are [linearly separable](https://en.wikipedia.org/wiki/Linear_separability).</span></span>

<span data-ttu-id="d9305-130">一般に、線形アルゴリズムはスケーラブルで高速であり、トレーニング コストが低コストで、予測も低コストです。</span><span class="sxs-lookup"><span data-stu-id="d9305-130">In general linear algorithms are scalable and fast, cheap to train, cheap to predict.</span></span> <span data-ttu-id="d9305-131">これらは、特徴の数と、おおよそのトレーニング データ セットのサイズによってスケールします。</span><span class="sxs-lookup"><span data-stu-id="d9305-131">They scale by the number of features and approximately by the size of the training data set.</span></span>

<span data-ttu-id="d9305-132">線形アルゴリズムは、トレーニング データに対して複数回実行されます。</span><span class="sxs-lookup"><span data-stu-id="d9305-132">Linear algorithms make multiple passes over the training data.</span></span> <span data-ttu-id="d9305-133">データセットがメモリに収まる場合は、トレーナーを追加する前に ML.NET パイプラインに[キャッシュ チェックポイント](xref:Microsoft.ML.LearningPipelineExtensions.AppendCacheCheckpoint*)を追加すると、トレーニングが高速になります。</span><span class="sxs-lookup"><span data-stu-id="d9305-133">If your dataset fits into memory, then adding a [cache checkpoint](xref:Microsoft.ML.LearningPipelineExtensions.AppendCacheCheckpoint*) to your ML.NET pipeline before appending the trainer, will make the training run faster.</span></span>
